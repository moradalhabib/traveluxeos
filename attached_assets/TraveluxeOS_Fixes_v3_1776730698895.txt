The current hotel and apartment commission model is wrong. Here is how it actually works:

CORRECT MODEL:
- Third party supplier (e.g. HotelBeds, Ratehawk, apartment owner) charges TVL a base cost
- TVL charges the client a higher price
- TVL's profit/commission = client price minus supplier cost
- Example: Supplier charges £300 for apartment → TVL charges client £350 → TVL earns £50

This is a MARKUP model, not a referral fee model. Fix the booking form and finance calculations for Hotel and Apartment bookings:

HOTEL BOOKING FIELDS (replace current arrangement fee model):
- Supplier cost (what HotelBeds / Ratehawk / supplier charges TVL) — manual input £
- Client price (what TVL charges the client) — manual input £
- TVL margin (auto-calculated: client price minus supplier cost)
- Payment method: Cash / Bank Transfer / Card
- Payment status: Paid / Unpaid / Partial

APARTMENT BOOKING FIELDS (same model):
- Supplier cost (what the apartment owner / supplier charges TVL) — manual input £
- Client price (what TVL charges the client) — manual input £
- TVL margin (auto-calculated: client price minus supplier cost)
- Nightly rate (for reference — manual input)
- Number of nights (auto-calculated from check-in and check-out)
- Payment method and status

Remove the "Arrangement Fees" tab from the Commissions page entirely — it no longer applies. Hotel and Apartment margins now flow into the Finance dashboard under TVL profit like all other services.

Update Finance and Profit tab to correctly include Hotel and Apartment margins in total TVL profit calculations.
